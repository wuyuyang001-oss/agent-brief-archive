StaminaBench: Stress-Testing Coding Agents over 100 Interaction Turns

选择原因：这是自上次已发布简报（2026-06-17）以来的新论文，直接评估 coding agent 在连续需求变更中的长期可靠性。它补足了单次任务成功率无法回答的关键问题：一个 agent 能稳定工作多少轮，以及模型、harness、测试反馈分别贡献了什么。

英文简介与对应中文

English

StaminaBench evaluates how long a coding agent can survive a sequence of valid, stateful change requests rather than whether it can solve one isolated task. Each run starts from a REST API implementation and applies up to 100 procedurally generated modifications inside an isolated environment. Across seven open or open-weight models and multiple agent harnesses, systems without a test-feedback loop fail after only a few turns; detailed feedback and retries improve endurance substantially, while the same model can differ by several-fold across harnesses. The benchmark therefore measures the joint reliability of the model, tool loop, context management, and recovery strategy.

中文对应翻译

StaminaBench 评估的不是 coding agent 能否完成一个孤立任务，而是它能在一系列合法、带状态的连续变更请求中坚持多久。每次运行从一个 REST API 实现开始，在隔离环境中依次施加最多 100 个程序化生成的修改。对 7 个开放或开放权重模型及多个 agent harness 的实验表明：没有测试反馈闭环时，系统通常只经过几轮就失败；详细反馈与重试能显著提高耐力，而同一模型搭配不同 harness 时，结果可相差数倍。因此，该 benchmark 衡量的是模型、工具循环、上下文管理与恢复策略的联合可靠性。

方法、结果与局限

核心方法在黑盒 HTTP 接口下，让 agent 构建 REST API，并依次处理 100 个由结构化动作空间生成的变更。20 个固定场景保证配置间可比；测试、agent 与服务均在隔离环境运行。
主要创新用“首次失败前连续通过的轮数”替代一次性解题率；将反馈粒度、重试预算、编程语言、场景采样器与 harness 作为可控变量；测试完全程序化生成，不依赖 LLM 判分。
关键结论无反馈时最佳组合平均仅通过 6.2 轮。启用 2 次重试后，GLM-5 + OpenCode 达到 57.0 ± 8.6 轮、25% 场景完整通过；详细反馈相对最小反馈可带来 6 至 12 倍提升。OpenCode 对 7 个模型均为最佳或统计上不劣于最佳，说明 harness 影响不能忽略。
可信度同一组 20 个场景用于配置间配对比较，并采用配对 Wilcoxon 检验及 Holm 校正；但部分差异在 n=20 下未达到显著性，且五次重复的配置均值仍可波动约 6 至 7 轮。
局限完整网格成本极高，未覆盖前沿闭源模型；仅 20 个场景，统计功效有限；任务集中于 REST API 服务，结论未必泛化到 GUI、移动端或大型遗留仓库。论文估算单个闭源配置可能耗资约 1.36 万至 2.27 万美元。
