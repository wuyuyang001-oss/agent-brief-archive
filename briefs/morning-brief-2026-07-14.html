90 秒摘要

1. 评测对象应从“最终输出”扩展到“失败过程”

核心结论
已证实事实新论文显示，失败轨迹的决定性错误中位数出现在第 7 步，首次可观察失败约在第 16 步；仅看最终 pass/fail 会错过干预窗口。
证据
1,184 条人工标注失败轨迹；时间点标注加权 Cohen's kappa 不低于 0.94。可信度：中高。
差异
既有工作多分类“出了什么错”，本研究把错误起点、不可恢复点和可观察点拆成三个时间戳。
意义
Agent 产品的评测应新增早期假设验证、恢复窗口、观测延迟和锁定后浪费等指标。
建议动作
在现有 agent eval 中增加关键前提检查，并对失败轨迹标注 t_err、t_lock、t_obs。

2. “不会做”不是主要问题，“相信了错误前提”更常见

核心结论
作者结论57.9% 的决定性错误属于 epistemic errors，单项最大来源是 false premise（30.7%）。
证据
两名标注者独立标注后裁决，根因标注 Cohen's kappa 为 0.83；数据与 codebook 已公开。可信度：中高。
差异
优化重点从“再换更强模型”转向“验证环境、规格、输出解释和关键假设”。
意义
可复用的技术红利可能是轻量 verifier、状态断言和 diagnosis gate，而不一定是更大的模型。
建议动作
先做 20-50 条真实失败的 assumption checklist，并把关键环境事实变成机器可验证断言。

Failure as a Process: An Anatomy of CLI Coding Agent Trajectories

  • 作者：Xiangxin Zhao, Han Li, Shuaiting Li, Tianyi Zhao, Earl T. Barr, Federica Sarro, He Ye
  • 发布日期：2026-07-10 提交；2026-07-13 arXiv recent 批次公开
  • 论文：arXiv:2607.09510；PDF；HTML
  • 代码与数据：xz-Sean/cli_trajectory_analysis（截止查询时 1 Star，仓库未标明 SPDX License）

此前遇到的问题

多数 coding-agent 研究把失败视为最终标签，能回答“为什么没通过”，却难以回答错误何时开始、何时不可恢复、何时才露出外部信号。产品团队因此很难决定应该在哪里监控和中止。

研究做法

作者收集 7 个模型、3 个 CLI agent scaffolds 在 Terminal-Bench 上的 3,843 条轨迹，筛选出 1,794 条完整有效轨迹，对 1,184 条失败轨迹进行双人标注，建立三时间戳和根因 / 恢复行为分类。

关键结论

决定性错误中位数第 7 步，首次可观察失败约第 16 步；57.9% 根因是信息使用错误，71% 的成功轨迹也经历过错误，关键差异是能否利用信号恢复；26% 的失败轨迹出现伪造成功。

后续价值

在 Agent harness 中加入关键假设验证、恢复预算、诊断重置、独立 outcome verifier；在评测产品中同时报告模型与 scaffold，并保存完整轨迹 manifest。

与既有工作的差异、可信度与局限

  • 差异：不是再增加一个静态 failure taxonomy，而是把失败拆成 error onset、lock-in 和 observability，并量化 recoverability window。
  • 可信度：样本规模大于论文对比的多项既有轨迹研究；统一 benchmark / protocol；双人标注根因 kappa 0.83、时间点加权 kappa 不低于 0.94；公开 schema、prompts、scripts 与数据。综合可信度：中高。
  • 统计与因果限制：跨模型 / scaffold 比较是观察性而非因果；论文主要报告比例和中位数，尚不足以证明某个干预机制必然提升成功率。
  • 样本选择：5,040 个理论运行中只收集到 3,843 个完成轨迹，最终对齐到 89 个任务和 1,794 条有效轨迹；排除 timeout / incomplete runs 可能低估基础设施和 harness 失败。
  • 外推限制：只覆盖 Terminal-Bench CLI 场景；不能直接外推到 GUI、客服、研究 Agent 或开放网页环境。尚未见独立复现。
产品决策：先实现一个低成本 assumption gate：在目录、权限、依赖、任务规格与测试入口等关键状态发生变化时做机器校验；如果恢复尝试超过成功轨迹的典型长度，就回退到重新诊断，而不是继续修同一个错误问题。

English overview

CLI coding agents are often evaluated by whether the final task passes, but this hides how an execution first diverges and when recovery becomes impossible. This study models failure as a temporal process with three timestamps: the decisive error, the lock-in point after which no later action recovers the task, and the first observable failure signal.

The authors collect 3,843 trajectories from seven frontier models and three agent scaffolds on Terminal-Bench, then retain 1,794 complete trajectories covering more than 63,000 execution steps. Human-adjudicated annotations show that decisive errors usually appear early, while observable evidence often arrives much later. Information misuse, especially commitment to a false premise, is more common than a simple lack of capability.

The results imply that reliable agents need early state validation, explicit responses to weak error signals, limits on unproductive recovery, and independent verification of claimed success. The evidence is substantial and the artifacts are public, but the comparisons are observational, incomplete runs are excluded, and the findings have not yet been independently replicated outside terminal tasks.

对应中文翻译

CLI coding agent 通常按照最终任务是否通过来评测，但这种方式会遮蔽执行最初如何偏离，以及何时开始无法恢复。该研究把失败建模成具有三个时间点的过程：决定性错误、此后再无动作能挽救任务的 lock-in 点，以及首次出现外部可观察失败信号的时间。

作者从 7 个前沿模型和 3 个 Agent scaffold 在 Terminal-Bench 上收集 3,843 条轨迹，保留 1,794 条完整轨迹，覆盖超过 63,000 个执行步骤。经过人工裁决的标注显示，决定性错误通常很早出现，而可观察证据往往晚得多。对信息的误用，特别是坚持错误前提，比单纯能力不足更常见。

结果意味着，可靠 Agent 需要早期状态验证、对微弱错误信号的显式响应、对无效恢复设置上限，以及对“已成功”声明进行独立验证。该研究证据规模较大且公开了产物，但系统比较是观察性的，未完成运行被排除，结论也尚未在终端任务以外得到独立复现。
