90 秒摘要

今天最值得保留的不是更多榜单，而是一个评测原则：任务子集只有在能维持完整 benchmark 的决策时才有意义。

评测方法 · 新出现“跑了多少任务”不是有效的评测充分性定义

核心结论
作者结论在 0 个百分点胜出阈值下，AppWorld 只需 15% 任务便首次满足全部目标，tau-bench 需 25%，SWE-bench Verified 需 90%，SWE-bench Lite 到 95% 仍未满足。
证据 / 可信度
复放 SWE-bench、AppWorld、tau-bench 的公开任务级记录，代码、固定种子与 manifest 已开放；但只有单作者、尚无独立复现，结论只覆盖既有完成记录。可信度：中。
与既有工作的差异
不把子集分数相关性当作充分性，而是同时要求决策误差、任务组覆盖和“无法下结论”比例达标。
产品 / 评测意义
评测预算可以缩减，但缩减比例必须针对 benchmark、胜出阈值和覆盖规则校准，不能固定成“抽 25%”。
建议动作
把现有产品 eval 改成“阈值 + 分层覆盖 + unresolved 输出”，先用历史全量运行回放一条预算曲线。

Agent 产品 · 官方视频Codex 正把编码、电脑操作、发布和协作压进同一工作面

核心结论

证据 / 可信度
官方 YouTube 视频与频道 RSS，发布时间 07:30 CST；属于产品演示，不是独立成功率评测。可信度：中。
与既有工作的差异
重点从“更会写代码”转向从执行、审阅到托管发布的端到端闭环。
产品 / 评测意义
单点 coding benchmark 不再覆盖产品价值，应加入多项目协调、浏览器验证、发布和回滚任务。

今日论文

How Many Tasks Are Enough for Agent Benchmark Decisions? A Replay Analysis of Public LLM Agent Benchmarks

作者：Wei-Jung Huang　提交：2026-07-14 04:36 UTC　会议：KDD 2026 Workshop on Agentic AI Evaluation and Trustworthiness

arXiv · HTML 全文 · PDF · 代码与复现包

选择理由：这是自昨日截止后新增、且直接影响 Agent 评测产品预算和决策可信度的论文。它回答的不是“子集分数与全量分数相关吗”，而是“子集是否支持与完整 benchmark 相同的系统比较结论”。

此前遇到的问题
Agent benchmark 成本高，团队会缩减任务，但仅报告任务比例无法说明是否仍能做出同样的产品选择。
研究做法
复放 SWE-bench Lite / Verified、AppWorld 和 tau-bench 完整任务级记录；比较随机、分层、coverage-aware、cheap-first 等 reveal policy。
关键结论
不存在通用“足够比例”。0 pp 阈值下，充分预算从 AppWorld 15% 到 SWE-bench Verified 90%；Lite 到 95% 仍不充分。
与既有工作的差异
把“决策误差 + 任务组覆盖 + unresolved 比例”作为联合门槛，并允许输出“暂不下结论”。
可信度
中。552 个 Lite 有向比较、156 个 Verified、306 个 AppWorld、12 个 tau-bench；固定种子、代码和派生表开放。单作者、无独立复现，tau-bench 比较数较少。
局限
只重放固定完成记录，不能外推未来任务分布；分组依赖公开元数据；未覆盖 OSWorld、WebArena、AgentDojo 等；自适应选题可能改变预算。
后续待验证
把方法应用到真实产品失败集；验证基于语义失败簇的覆盖；把 token、延迟、交互次数和人类接管加入成本预算。
建议动作
先对内部完整历史运行做 replay，不直接把论文的 15% 或 90% 搬进生产。
对评测产品的直接启发：默认输出不应只有排行榜分数。至少要同时展示“胜出阈值、已覆盖任务组、错误上界、未决比较比例”，并在证据不足时给出继续跑哪些任务的建议。
对抗性审查：作者把完成 benchmark 当成参考决策，而完成 benchmark 自身不等于真实产品真值；任务之间也可能相关。该方法能降低“子集误导完整榜单”的风险，不能证明榜单能预测线上效果。
英文简介与逐段中文对应翻译

English

Agent benchmarks often compare two agents after all tasks have run, but costly evaluations make partial runs tempting. A task fraction alone does not show whether a partial run supports the same pairwise conclusion as the completed benchmark. This paper replays completed public task-level records from SWE-bench, AppWorld, and tau-bench. A partial budget is considered sufficient only when it preserves the completed benchmark's decision, covers required task groups, and leaves no more than a target fraction of comparisons unresolved. Under a strict zero-percentage-point threshold, AppWorld first meets all targets at 15%, tau-bench at 25%, and SWE-bench Verified at 90%; SWE-bench Lite does not meet all targets by 95% under the primary coverage rule. The study argues that partial-evaluation reports must specify the improvement threshold, task-selection policy, coverage rule, decision rule, and allowed unresolved rate.

中文对应翻译

Agent benchmark 通常在所有任务完成后比较两个 Agent，但高昂成本使部分运行很有吸引力。仅有任务比例不能说明部分运行是否支持与完整 benchmark 相同的成对结论。本文复放 SWE-bench、AppWorld 和 tau-bench 已完成且公开的任务级记录。只有当部分预算保持完整 benchmark 的决策、覆盖要求的任务组，并把未决比较控制在目标比例内时，才被视为充分。在严格的 0 个百分点阈值下，AppWorld 首次在 15% 达到全部目标，tau-bench 为 25%，SWE-bench Verified 为 90%；在主要覆盖规则下，SWE-bench Lite 到 95% 仍未满足全部目标。研究主张，部分评测报告必须明确改进阈值、任务选择策略、覆盖规则、决策规则和允许的未决比例。
