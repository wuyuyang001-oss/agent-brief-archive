评测方法 · 新出现领域 Agent 评测需要结构、执行与安全语义三种证据

核心结论
作者结论Alipay-PIBench 的静态检查、Integration/E2E 与 LLM 辅助领域审查会暴露不同失败；Basic 静态 RPR 均值 92.79%，Integration 仅 73.16%，Advanced E2E 97.03% 但领域审查 61.05%。
证据 / 可信度
18 个任务、9 类支付产品、6 个模型，代码与 benchmark 已开放；但由平台方提出、无独立复现，LLM judge 也可能有噪声。可信度：中。
与既有工作差异
不把“代码通过测试”当作最终正确，而是区分初始支付闭环与风控加固，并把 rubric 映射到确定性检查和语义审查。
产品 / 评测意义
Agent 产品的真实价值在跨组件交付和业务状态一致性；评测产品应能定位失败在哪一层，而不是只给一个总分。
建议动作
把内部 eval 的每条 rubric 映射为静态、单元、集成、E2E 或领域审查，并要求至少一个可执行信号支撑关键安全项。

Alipay-PIBench: A Realistic Payment Integration Benchmark for Coding Agents

作者：Shiyu Ying、Xuejie Cao、Yingfan Ma、Yuanhao Dong、Wenyu Chen、Bowen Song、Lin Zhu（Ant Group）　提交：2026-07-16 05:08 UTC

arXiv · HTML 全文 · PDF · 代码 / Benchmark

选择理由：这是上次运行后新增，且把 Agent 产品、领域 Skill 和评测方法放进同一个配对实验的论文；对构建 Agent 评测产品的价值高于今天其他只给榜单的论文。

此前遇到的问题
支付集成不是加一个 API：必须选择产品、守住凭证与签名边界、协调前后端、验证异步通知，并保持支付与业务状态一致。
研究做法
9 个产品项目 × Basic/Advanced 两阶段，共 18 个 repository-level 任务；rubric 派生静态、单元、集成、E2E 和 LLM 辅助审查，并做有/无支付 Skill 配对。
关键结论
with-skill 平均 RPR 为 68.58%–91.37%；Skill 平均 +10.31 pp，101/108 单元提升；Basic +11.27 pp、Advanced +9.35 pp。
与既有工作差异
把功能闭环与风控加固分开，并用多种证据定位“代码看起来完整但工作流或领域安全仍失败”。
可信度
中。任务和代码开放、配对设计清晰、评测信号多元；但每格缺少重复试验与置信区间，无独立复现，平台方自建 benchmark 有利益与覆盖偏差。
局限
仅覆盖 Alipay 产品和指定 Claude Code 版本；LLM judge 可能误判；真实资金环境、长时异常、外部依赖漂移与攻击场景覆盖有限。
后续待验证
跨支付平台、跨 Agent harness、重复随机试验、真实 sandbox 交易、故障注入和独立审计。
建议动作
把同样的“领域 Skill A/B + 多层 rubric”迁移到你的核心 Agent 产品任务，并把人工接管与成本纳入 RPR。
产品机会：把通用 Agent benchmark 变成“领域交付验证器”：允许企业上传业务 rubric，自动拆成静态、执行、E2E 与语义检查，输出失败定位、Skill 干预增益和上线门槛。
对抗性审查：101/108 提升很醒目，但若每个条件只有一次运行，Agent 随机性和运行中断可能放大差值；RPR 也由平台自定义 rubric 决定。采用前应重复运行、盲审关键项，并用真实业务事故反向校准 rubric。
英文简介与逐段中文对应翻译

English

Alipay-PIBench evaluates coding agents on realistic repository-level payment integration rather than isolated code generation. It contains nine product-specific projects and 18 task instances split into Basic functional completion and Advanced risk-aware hardening. Scenario rubrics are translated into static checks, unit, integration and end-to-end tests, plus rubric-aligned LLM-assisted review. Across six models, mean rubric pass rate with the domain skill ranges from 68.58% to 91.37%. Compared with matched runs without the skill, the skill improves mean RPR by 10.31 percentage points. The diagnostic results show that source-level completion, executable behavior and payment-domain safety can diverge, so no single evaluator is sufficient. The evidence is useful but platform-specific, and it lacks independent replication and repeated-trial uncertainty.

中文对应翻译

Alipay-PIBench 评估的是现实仓库级支付集成，而不是孤立代码生成。它包含 9 个产品项目和 18 个任务实例，分为 Basic 功能闭环与 Advanced 风险感知加固。研究把场景 rubric 转换为静态检查、单元测试、集成测试、端到端测试，以及与 rubric 对齐的 LLM 辅助审查。在 6 个模型中，使用领域 Skill 后的平均 rubric 通过率为 68.58%–91.37%；与无 Skill 的配对运行相比，Skill 使平均 RPR 提升 10.31 个百分点。诊断结果显示，源码层完成度、可执行行为和支付领域安全可能彼此背离，因此单一评测器并不足够。这些证据有用但局限于特定平台，且缺少独立复现和重复试验的不确定性。
