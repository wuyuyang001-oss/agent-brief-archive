评测方法 · 新出现

评测器也要随策略进化

已证实：DynamicRubric 根据同一提示下的候选响应集动态生成加权二元 rubric，再由冻结 verifier 判定；8B 生成器在多组 evaluator benchmark 上超过更大的静态方法。

差异：不再给每个回答一个固定绝对分，而是针对“当前难以区分的一组回答”寻找可分辨标准。

产品意义：Agent 评测产品应支持版本化 rubric、候选集条件化和判分漂移监控。

建议动作：用你自己的 Agent 失败对生成动态 rubric，与固定 rubric 做盲评一致性比较。

Agent 产品 · 关键版本

Claude Code 把 review 移到后台子 Agent

已证实：v2.1.218 将 /code-review 放入后台，补强 MCP 连接错误、workspace trust、session lineage、自动权限分类与无障碍支持。

差异：多 Agent 价值从“更多并发”转向隔离上下文、保留主对话和明确权限边界。

建议动作：评测后台 review 是否遗漏上下文、是否可追溯、以及主 Agent 退出时任务是否完整收敛。

今日论文

Co-Evolving LLM Evaluators and Policies via DynamicRubric

Beining Wang 等 · 清华大学 / 微信 · v1 2026-07-22，v2 2026-07-23 · arXiv · HTML · 未发现作者公开代码仓库

此前问题：策略变强后，同一提示的候选回答质量越来越接近，固定 reward model 的相对分差塌缩，策略优化获得弱或错误信号。

研究做法：DR-Generator 针对当前候选集生成带权二元 rubric；冻结的 DR-Verifier 逐项判定，再聚合为回答分数；generator 与 policy 交替更新。

关键结论：作者报告 8B DynamicRubric 在 evaluator、开放生成、数学与代码任务上均有增益，并已部署于微信搜索。

与既有工作差异：静态 rubric 只依赖 prompt，DynamicRubric 同时看 prompt 与当前 response set，优化目标强调候选间可分辨性。

可信度：中等偏高 多 benchmark、跨 verifier/backbone 与生产部署提供多层证据；但训练和评测主要由作者体系完成，线上指标不透明，无独立复现。

局限：额外 generator/verifier 调用增加训练延迟；冻结 verifier 决定可用监督边界；动态 rubric 可能奖励“容易分割”而非真正重要的质量维度。

待验证：在工具调用轨迹、computer-use 安全行为和长程任务上，动态 rubric 是否仍与人工偏好及真实成功率一致。

展开英文简介与对应中文翻译

English. As policies improve, their sampled responses become harder to distinguish, so static evaluators often produce collapsed or misleading score gaps. DynamicRubric conditions evaluation criteria on the current response set: a trainable generator creates weighted binary rubrics, a frozen verifier applies them, and the resulting scores supervise policy updates. The paper reports stronger evaluator discrimination and policy gains with 8B backbones, including transfer to verifiable reasoning and coding tasks. Its main risks are training overhead, dependence on verifier scope, and the absence of independent replication.

中文。 随着策略能力提升，其采样回答越来越难区分，静态评测器往往产生塌缩或误导性的分差。DynamicRubric 让评测标准依赖当前候选集：可训练生成器产生带权二元 rubric，冻结 verifier 执行判定，聚合分数再监督策略更新。论文报告 8B 骨干下更强的评测区分能力和策略增益，并迁移至可验证推理与代码任务。主要风险是训练开销、对 verifier 能力边界的依赖，以及缺少独立复现。
