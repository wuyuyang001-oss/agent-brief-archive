90 秒摘要

Agent 架构 · 新出现

长程记忆可以先从“完整日志 + 代码检索”开始

已证实：PRO-LONG 不做摘要压缩，也不依赖向量库，而是保存所有观察、动作和结果，让 coding agent 用 grep、Python 和自写脚本检索。

证据：完整 ARC-AGI-3 公共游戏集上，相对基础 coding agent 平均提升 18.0 个百分点；最高 76.1% pass@1，token 少 4.2–5.8 倍。

差异：把“记住什么”的智能从写入阶段移到读取阶段，降低早期摘要丢失未来关键细节的风险。

建议动作：先为现有 Agent 增加 append-only 事件日志和可执行查询，再决定是否需要向量记忆。

Agent 产品 · 官方发布

ChatGPT Health 展示了高风险连接器的产品范式

官方发布：用户可连接 Apple Health 与支持的医疗记录；默认在使用连接数据前请求许可，敏感外发动作增加确认，连接数据不用于基础模型训练。

差异：连接器不只是“多一个数据源”，而是独立权限、数据生命周期、记忆边界和外发防护的组合。

产品意义：Agent 平台应把数据域、目的限制、临时授权和撤销删除做成一等能力。

建议动作：以 Health 的权限路径为参照，审查你的连接器是否区分读取、推理、记忆和外发。

评测方法 · 证据增强

长程 Agent 评测必须同时报告 pass@1、best@k 与成本

已证实：PRO-LONG 在同一游戏的独立运行间方差明显；论文重算先前 harness 日志以统一 500-action 预算和评分。

差异：单次成功率无法说明探索覆盖率，best@k 又会放大预算优势。

建议动作：固定动作预算、重复次数和总成本，并把 pass@1 与 best@k 并列展示。

产品机会 · 编辑推断

Agent 记忆产品的价值可能在可审计查询，而非“更聪明的摘要”

编辑判断：当日志达到 10 万至 32 万行仍可通过程序化检索利用时，可观测、查询 DSL、重放与轨迹验证可能比又一套 embedding memory 更接近刚需。

创业假设：面向 Agent 的“事件仓库 + 查询 + 回放 + eval”可同时服务调试、记忆、安全审计和产品分析。

今日论文

PRO-LONG: Programmatic Memory Enables Long-Horizon Reasoning

Alexis Fox、Junlin Wang、Paul Rosu、Bhuwan Dhingra · Duke University · v1 2026-07-22，v2 2026-07-23 · arXiv · HTML · 代码与日志

此前问题：长程 Agent 的摘要会丢失未来才显得关键的细节；保留全部上下文又昂贵且易受 context rot 影响。

研究做法：harness 将每次观察、动作、计划与结果写入结构化日志；Agent 用 shell、正则和 Python 程序按需读取，不使用专用向量数据库或训练过的 memory writer。

关键结论：25 个 ARC-AGI-3 公共游戏上，平均提升 18.0pp；最高 76.1% pass@1，4.2–5.8 倍更少 token；Fable 5 达到 97.4% best@2，总成本 1,750 美元。

与既有工作差异：WorldModeler、Schema 等加入定制 world model、MCP 工具或多 Agent；PRO-LONG 只对标准 Codex/Claude Code 增加约 30 行提示和完整日志。

可信度：中等偏高 使用完整公共游戏集、统一预算重算先前日志、报告 bootstrap 区间并公开代码/轨迹；但仅一个游戏类 benchmark，部分先前方法缺少完整重复运行。

局限：新模型可能见过公开 benchmark；best@k 受预算驱动；日志无界增长带来存储、隐私和查询安全问题；没有真实企业长程任务的独立复现。

待验证：在 coding、computer-use 与业务流程中，日志规模、检索延迟、敏感信息隔离和故障恢复是否仍保持收益。

展开英文简介与对应中文翻译

English. Long-horizon agents must preserve details whose relevance may emerge only much later. PRO-LONG uses programmatic memory: every observation, action, and outcome is appended to a structured log, while the coding agent retrieves evidence with regular expressions, shell tools, and Python. On the 25 public ARC-AGI-3 games, the method substantially improves standard coding agents and competes with specialized harnesses at far lower token cost. The evidence is strengthened by released code and logs, matched action budgets, repeated runs, and confidence intervals, but remains limited to one public benchmark with high run-to-run variance.

中文。 长程 Agent 必须保留那些可能在很久以后才显现价值的细节。PRO-LONG 使用程序化记忆：每次观察、动作和结果都追加到结构化日志，coding agent 再通过正则、shell 工具和 Python 检索证据。在 25 个公开 ARC-AGI-3 游戏上，该方法显著增强标准 coding agent，并以更低 token 成本接近或超过专用 harness。公开代码与日志、统一动作预算、重复运行和置信区间增强了证据，但结论仍局限于一个公开 benchmark，且运行方差很高。
